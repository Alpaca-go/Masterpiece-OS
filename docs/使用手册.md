# 使用手册

## 推荐工作流

1. 在 `projects/` 下创建项目文件夹并放入全部原始素材。
2. 实际查看每张视觉图片，不得仅依据文件名、OCR、尺寸或元数据判断。
3. 在 `design-factory.json` 写入 Brand Lock、视觉核验结果、Benchmark 与 Creative Reasoning。
4. 执行 Fast Mode，检查品牌设计推理和 Chat 生图任务包。
5. 确认所有“待确认”信息后再执行图片任务。
6. 只有需要知识审核或成长评审时才切换 Review/Research Mode。

## Fast Mode（默认）

```bash
npm run analyze -- --project "项目名称"
```

需要在线对标候选时添加 `--online`。Fast Mode 只生成：

- `01-项目分析报告.md`
- `02-Chat生图任务包.md`

它不会运行 Knowledge Analysis、Design Review 或成长历史写入。

## Review Mode

```bash
npm run analyze -- --project "项目名称" --mode review
```

Review Mode 在两份核心文件之外增加：

- `03-Knowledge-Review.md`
- `04-Design-Review.md`

并在本地 `history/reviews/` 保存成长记录。可使用 `--history-dir <目录>` 指定其他位置。

## Research Mode

```bash
npm run analyze -- --project "项目名称" --mode research
```

Research Mode 用于知识研究和开发验证。它可以读取 `--knowledge-dir <目录>`，但永远不会自动修改 Approved Knowledge。

## Creative Reasoning

`creativeReasoning` 包含：

- `positioning`：品牌类型与判断依据
- `keywords`：关键词及原因
- `temperament`：品牌气质与依据
- `visualDNA`：色彩、构图、留白、摄影、包装、工艺、必须保留与必须避免
- `photographyLanguage`：光线、镜头、材质与氛围
- `creativeDirection`：所有新增图片共同遵循的一句话方向
- `designRisks`：问题、原因和避免方式

如果没有完成逐张视觉核验，系统会在报告中标记核验未闭环，构图、留白、摄影和工艺等信息保持待确认。

## Chat 任务包

任务包第一层是“品牌设计意图”，第二层是“图片任务”。所有任务默认继承第一层，因此执行时无需重新分析品牌，也不要在每张图中改写品牌方向。

## 直接路径模式

旧版直接路径仍然兼容：

```bash
node bin/design-factory.js analyze ./my-brand --mode fast
```

## 调试

添加 `--debug` 会额外生成 `design-factory-result.json`。正式使用不要开启调试输出。

## 支持范围

| 格式 | 能力 |
|---|---|
| ZIP | 读取中央目录、文件名与数量，不解压 |
| PDF | 页数、基础元数据与可见文本线索 |
| PPTX/POTX/PPSX | 页数、幻灯片文本、主题色与字体 |
| 旧版 PPT | 纳入清单，暂不深度解析 |
| PNG/JPEG/GIF/WebP/SVG/BMP | 格式和尺寸；SVG 额外提取颜色与文本 |
| MD/TXT/JSON/CSV | 文本线索，最多读取前 200KB |

文件解析能力不等于视觉核验。像素画面的设计判断必须由具备视觉能力的模型或人员逐张完成，再写入项目配置。

## 常见问题

### 为什么默认只有两份文件？

v3.0 将高质量设计沟通设为默认目标，Fast Mode 跳过与图片规划无关的完整评审。需要四份文件时使用 `--mode review`。

### 为什么报告写着“视觉核验未闭环”？

只有 `visualInspection.verified=true` 且 `inspectedImageCount` 不少于实际图片数时才视为闭环。请先逐张查看图片并补充真实画面发现。

### 联网失败会阻塞吗？

不会。系统会记录失败原因并回退到内置策展案例；生产项目仍应人工核验案例的同类型与同定位程度。

### 重复运行安全吗？

安全。`outputs/` 不会被回扫；Fast Mode 会清理旧的 03/04 文件，避免把过期 Review 当成本次结果。
