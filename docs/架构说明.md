# 架构说明

流水线分为八层：

1. `project-paths` 约束项目必须位于 `projects/` 的直接子目录，并阻止路径穿越和符号链接逃逸。
2. `project-selector` 发现本地项目；单项目自动选择，多项目要求明确指定。
3. `project-initializer` 在分析前创建固定目录并安全整理素材，冲突时不覆盖、不移动。
4. `inventory` 递归清点文件，并调用格式解析器。
5. `analyze` 从素材证据和项目配置生成 Brand Lock、对标、缺图和图片规划。
6. `knowledge-analysis` 生成候选、只读加载 Approved Rule，并执行分类、对比、健康度和优先级分析。
7. `pipeline` 组织阶段并形成唯一结构化结果。
8. `report` 从同一结果渲染设计报告、三个规范交付文件和 JSON，防止跨文档不一致。

配置优先级为“人工配置 > 可解析素材证据 > 行业默认建议”。联网结果只作为对标候选，不会自动写入 Brand Lock。

解析器不把 ZIP 内容释放到磁盘，降低路径穿越风险。单个文件解析失败会保留在素材清单并记录警告，不中断其余项目分析。

Knowledge 写入边界是单向的：`knowledge/approved/ → Knowledge Analysis → outputs/`。流水线没有任何写回 Approved Rule 的代码路径。
