# 架构说明

Design Factory OS v3.0 的核心链路为：

```text
Brand Lock → Benchmark Analysis → Creative Reasoning → Image Planning → Chat Package
```

流水线分为十一层：

1. `project-paths`：限制项目位于 `projects/` 的直接子目录，阻止路径穿越和符号链接逃逸。
2. `project-selector`：发现本地项目；单项目自动选择，多项目要求明确指定。
3. `project-initializer`：创建固定目录并安全整理素材，冲突时不覆盖、不移动。
4. `inventory`：递归清点文件并调用格式解析器。
5. `analyze`：根据素材证据和配置生成 Brand Lock、Benchmark、缺图与图片规划。
6. `creative-reasoning`：在图片规划前建立品牌定位、关键词、气质、视觉 DNA、摄影语言、创意方向和 Design Risks。
7. `knowledge-analysis`：仅在 Review/Research Mode 生成候选并只读对比 Approved Rule。
8. `design-review`：仅在 Review/Research Mode 生成专业评审、能力雷达和行动建议。
9. `growth-engine`：仅在 Review/Research Mode 读取和保存本地成长历史。
10. `pipeline`：按运行模式组织阶段并形成唯一结构化结果。
11. `report`：Fast Mode 渲染两份核心报告；Review/Research Mode 渲染四份编号报告；不新增其他 Markdown。

## Creative Reasoning 数据边界

配置优先级是“人工逐张视觉核验 > 已确认 Brand Lock 与 Benchmark > 安全回退”。

`visualInspection.verified` 只有在核验数量覆盖全部图片时才生效。未闭环时，系统不会把文件名、OCR、尺寸或元数据推断成构图、留白、摄影或工艺事实，而是保留“待确认”并生成对应 Design Risk。

## 模式边界

| 模式 | 核心分析 | Knowledge | Design Review | 历史写入 | Markdown |
|---|---|---|---|---|---:|
| fast | 是 | 否 | 否 | 否 | 2 |
| review | 是 | 只读分析 | 是 | 是 | 4 |
| research | 是 | 只读分析 | 是 | 是 | 4 |

Fast Mode 是默认模式，避免每个项目都重复运行完整评审。Research Mode 不代表授权写入正式 Knowledge；任何正式维护仍需人工审核并另行执行。

## 输出与安全

解析器不会把 ZIP 内容释放到磁盘。单个文件解析失败会记录警告但不中断其他素材。

Knowledge 数据流保持单向：

```text
knowledge/approved/ → Knowledge Analysis → outputs/
```

成长历史仅用于 Review/Research Mode 的趋势计算，不会反向修改评分规则、Knowledge、Rule、Prompt 或 Template。
