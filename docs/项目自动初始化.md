# Projects 自动初始化

## 用户流程

把一个以项目名称命名的文件夹放入仓库的 `projects/`，素材可以暂时散放在该项目根目录。然后运行：

```bash
npm run analyze -- --project "项目名称"
```

当 `projects/` 中只有一个项目时，可以直接运行 `npm run analyze`。存在多个项目时必须使用 `--project` 明确选择，系统不会猜测。

## 整理规则

分析前会确保项目具有固定结构：

```text
projects/项目名称/
├─ input/
├─ outputs/
└─ design-factory.json（可选）
```

项目根目录中的普通文件和子目录会整体移入 `input/`，嵌套结构保持不变。`input/`、`outputs/`、`design-factory.json`、`.gitkeep`、隐藏文件和常见系统文件保留原位。旧版 `inputs/` 中的内容会安全迁移到标准 `input/`。

## 安全行为

- 移动前检查全部目标；任一同名冲突都会停止，且不移动任何素材。
- 永不覆盖已有文件，重复运行保持幂等。
- 项目名称只能指向 `projects/` 的直接子目录，拒绝绝对路径、`../`、符号链接逃逸等越界访问。
- 移动过程中若发生文件系统错误，会报告已移动项和待处理项，便于人工恢复。

## Git 边界

Git 只跟踪 `projects/.gitkeep`。`projects/` 下所有真实项目、素材和分析输出均由 `.gitignore` 排除。初始化和分析命令不会自动执行 `git add`、`git commit` 或 `git push`。
